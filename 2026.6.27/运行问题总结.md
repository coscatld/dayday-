# 运行问题总结：问题、解决办法与知识点

## 一、遇到的问题

### 1. 命令行无法直接运行 Java 和 Maven

在命令行执行：

```powershell
java -version
mvn -version
```

结果提示：

```text
无法将“java”项识别为 cmdlet、函数、脚本文件或可运行程序的名称
无法将“mvn”项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

原因是本机虽然安装了 JDK 和 Maven，但没有配置到系统 `PATH` 环境变量中。

本机实际可用的 JDK 路径是：

```text
C:\Users\lenovo\.jdks\ms-17.0.16
```

本机可用的 Maven 路径是：

```text
C:\Users\lenovo\.m2\wrapper\dists\apache-maven-3.9.14-bin\1cb7fhup6b5n3bed6kckbrnspv\apache-maven-3.9.14\bin\mvn.cmd
```

### 2. 项目启用了 dev 环境，但缺少 application-dev.yml

`src/main/resources/application.yml` 中配置了：

```yaml
spring:
  profiles:
    active: dev
```

这表示项目启动时会加载 `dev` 环境配置。

但原项目中只有：

```text
application.yml
```

没有：

```text
application-dev.yml
```

这会导致本地开发环境下缺少数据库、Redis、AI 等关键配置，项目很容易启动失败。

### 3. 阿里云 OSS 密钥为空导致启动失败

项目中 `AliOssConfig` 会在 Spring Boot 启动时创建 OSS 客户端：

```java
new OSSClientBuilder().build(
        aliOssProperties.getEndpoint(),
        aliOssProperties.getAccessKeyId(),
        aliOssProperties.getAccessKeySecret()
);
```

但 `application.yml` 中默认配置是：

```yaml
alioss:
  access-key-id: ${ALIOSS_ACCESS_KEY_ID:}
  access-key-secret: ${ALIOSS_ACCESS_KEY_SECRET:}
```

如果没有配置环境变量，`access-key-id` 和 `access-key-secret` 就是空字符串。

启动时报错：

```text
Access key id should not be null or empty.
```

这个错误会导致 Spring 容器初始化失败，项目无法启动。

### 4. Chroma 服务未启动，但不是致命错误

启动日志中出现：

```text
Chroma 不可用，向量检索功能暂无法使用
```

这是因为本地没有启动 Chroma 向量数据库服务。

不过代码中已经做了异常处理，所以这只是警告，不会阻止项目启动。

### 5. 中文日志在 PowerShell 中显示乱码

部分中文注释和日志在 PowerShell 中看起来像乱码，是因为终端默认编码与文件 UTF-8 编码不一致。

源码本身没有损坏，使用 UTF-8 方式读取时内容正常。

## 二、解决办法

### 1. 新增 application-dev.yml

新增文件：

```text
src/main/resources/application-dev.yml
```

用于补充 `dev` 环境下的本地开发配置：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://${MYSQL_HOST:localhost}:${MYSQL_PORT:3306}/${MYSQL_DATABASE:wudong_demo}?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: ${MYSQL_USERNAME:root}
    password: ${MYSQL_PASSWORD:root}

  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:}
      database: ${REDIS_DATABASE:0}

mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl

ai:
  llm:
    api-key: ${DASHSCOPE_API_KEY:sk-local-placeholder}
  embedding:
    api-key: ${DASHSCOPE_API_KEY:sk-local-placeholder}
  vector-store:
    base-url: ${CHROMA_BASE_URL:http://localhost:8000}
    collection-name: ${CHROMA_COLLECTION:wudong_kb}
```

这样项目启动时可以正确加载本地开发环境配置。

### 2. 修改 AliOssConfig，避免没有密钥时强制创建 OSS 客户端

原来的问题是：不管有没有 OSS 密钥，项目启动时都会创建 `OSS` Bean。

修复后：只有在配置了 `alioss.access-key-id` 和 `alioss.access-key-secret` 时，才创建 OSS 客户端。

核心修改：

```java
@Bean
@ConditionalOnExpression("T(org.springframework.util.StringUtils).hasText('${alioss.access-key-id:}') && T(org.springframework.util.StringUtils).hasText('${alioss.access-key-secret:}')")
public OSS ossClient() {
    return new OSSClientBuilder().build(
            aliOssProperties.getEndpoint(),
            aliOssProperties.getAccessKeyId(),
            aliOssProperties.getAccessKeySecret()
    );
}
```

这样开发环境没有 OSS 密钥时，项目也可以正常启动。

### 3. 修改 FileUploadUtil，支持本地文件上传回退

原来的上传逻辑只支持 OSS：

```java
ossClient.putObject(...);
```

修复后：

如果配置了 OSS，则上传到阿里云 OSS。

如果没有配置 OSS，则保存到本地 `uploads` 目录，并返回 `/files/xxx` 访问路径。

核心逻辑：

```java
OSS ossClient = ossClientProvider.getIfAvailable();
if (ossClient != null) {
    ossClient.putObject(aliOssProperties.getBucketName(), objectName, file.getInputStream());
    return "https://" + aliOssProperties.getBucketName() + "."
            + aliOssProperties.getEndpoint() + "/" + objectName;
}

Path uploadPath = Path.of(uploadDir).toAbsolutePath().normalize();
Files.createDirectories(uploadPath);
Files.copy(file.getInputStream(), uploadPath.resolve(objectName), StandardCopyOption.REPLACE_EXISTING);
return normalizeAccessPath(accessPathPrefix) + objectName;
```

### 4. 使用本机已有 JDK 和 Maven 进行构建

因为 `java` 和 `mvn` 没有配置到系统 `PATH`，所以可以临时指定路径执行。

编译打包命令：

```powershell

& 'C:\Users\lenovo\.m2\wrapper\dists\apache-maven-3.9.14-bin\1cb7fhup6b5n3bed6kckbrnspv\apache-maven-3.9.14\bin\mvn.cmd' -DskipTests package
```

运行命令：

```powershell
& 'C:\Users\lenovo\.jdks\ms-17.0.16\bin\java.exe' -jar 'C:\Users\lenovo\Desktop\str\dance-web-main\target\wudong-demo-0.0.1-SNAPSHOT.jar' --server.port=18080
```

## 四、相关知识点

### 1. Spring Boot Profile

`spring.profiles.active` 用来指定当前运行环境。

例如：

```yaml
spring:
  profiles:
    active: dev
```

会额外加载：

```text
application-dev.yml
```

如果激活了某个 profile，却没有提供对应配置，就可能导致数据库、Redis、第三方服务等参数缺失。

### 2. 环境变量占位符

Spring Boot 支持这种写法：

```yaml
username: ${MYSQL_USERNAME:root}
```

含义是：

如果系统环境变量 `MYSQL_USERNAME` 存在，就使用环境变量的值。

如果不存在，就使用默认值 `root`。

这种方式适合区分开发、测试、生产环境，避免把真实账号密码写死在代码里。

### 3. Bean 条件装配

有些组件只应该在满足条件时创建，比如 OSS 客户端必须有密钥才能创建。

可以使用条件注解控制 Bean 是否注册：

```java
@ConditionalOnExpression(...)
```

这样可以避免“某个外部服务没配置，整个项目都启动不了”的问题。

### 4. ObjectProvider

`ObjectProvider<T>` 可以用来注入“可选 Bean”。

例如：

```java
private final ObjectProvider<OSS> ossClientProvider;

OSS ossClient = ossClientProvider.getIfAvailable();
```

如果 Spring 容器中有 `OSS` Bean，就返回它。

如果没有，就返回 `null`，不会导致启动失败。

### 5. 本地文件上传回退

开发环境通常不一定有真实 OSS、COS、S3 等对象存储密钥。

更稳妥的设计是：

生产环境：上传到对象存储。

开发环境：保存到本地目录。

这样可以降低本地启动门槛，方便调试。

### 6. PATH 环境变量

命令行能否直接执行：

```powershell
java
mvn
```

取决于对应可执行文件所在目录是否加入了系统 `PATH`。

如果没加入，也可以通过完整路径执行。

长期建议把 JDK 的 `bin` 目录和 Maven 的 `bin` 目录加入系统环境变量。

### 7. PowerShell 编码问题

源码文件通常使用 UTF-8 编码。

如果 PowerShell 当前编码不是 UTF-8，中文注释或日志可能显示为乱码。

这通常是终端显示问题，不代表源码损坏。
